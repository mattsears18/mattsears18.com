# Privacy — data processor inventory

Canonical record of every third-party service that processes visitor data on `mattsears18.com`. Paired with the public-facing notice at [`/privacy`](https://mattsears18.com/privacy) (rendered from [`app/privacy/page.tsx`](./app/privacy/page.tsx)); that page is the disclosure for visitors, this file is the internal record that keeps the disclosure honest.

Update this file in the same PR that introduces (or removes) any service receiving visitor data. If unsure whether something counts as a processor, list it — over-disclosure is cheap, under-disclosure is the failure mode this file exists to prevent.

The site is a single-author personal/portfolio site with no analytics scripts, no cookies, no contact form on-site, and no accounts. The processor surface is intentionally small.

## Not processors (clarifications)

These look like they might be processors but aren't:

- **`next/font/google` (Inter, Source Serif 4, JetBrains Mono)** — declared in [`app/layout.tsx`](./app/layout.tsx) but served via Next.js's [self-hosting pipeline](https://nextjs.org/docs/app/api-reference/components/font). Font files are downloaded at build time and shipped from the site origin; no `fonts.googleapis.com` / `fonts.gstatic.com` request happens at page-load time. Google is not a runtime processor for visitors.
- **`@vercel/toolbar`** — a development-only overlay that is gated by `process.env.VERCEL_ENV !== 'production'` in [`app/layout.tsx`](./app/layout.tsx) (`showVercelToolbar` branch). It does not render for production visitors and is not a production processor.
- **Outbound links** — the site links out to GitHub, LinkedIn, and a small handful of other services. Once a visitor follows a link, the destination's own privacy policy applies; the site does not relay any data to those services on its own.
- **Web analytics** — none installed. No Google Analytics / Plausible / Fathom / PostHog / Umami / Vercel Analytics. If one is ever added it lands here in the same PR.

## Records of Processing Activities (GDPR Art. 30)

The site is well under the Art. 30(5) threshold (no employees processing on behalf of an organisation, no special-category data, no large-scale or systematic monitoring of data subjects), so a formal Art. 30 RoPA is not required. This file functions as a lightweight equivalent — when / if the threshold is ever crossed, it becomes the basis for a formal RoPA rather than starting from scratch.

## Change procedure

1. Adding a new dependency or SDK that sends data off-host? Add a row to **Active production processors**, with the same columns the existing rows use. Include the sub-processor link and the transfer mechanism.
2. Removing a service? Strike it out (or remove the row) in the same PR that removes the integration, and update [`app/privacy/page.tsx`](./app/privacy/page.tsx) if the public-facing disclosure mentioned it.
3. The [`/privacy`](./app/privacy/page.tsx) page is the visitor-facing summary; this file is the source of truth — keep them aligned. The PR template's contributor checklist surfaces this reminder when a new dependency lands.
